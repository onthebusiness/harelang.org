Hare is a statically typed programming language which compiles to machine code. It has no runtime, manual memory management, and it uses the System-V ABI. Hare is designed for systems programming, and is well-suited to writing operating systems, system tools, compilers, and other low-level, high-performance tasks.

Hare fits on a 3½" floppy disc, one of which you may purchase.

Hare runs on most Unix-like systems, and targets x86_64 and arm64 architectures. If you need additional architectures, they can be added to our backend in ≤2,000 lines of code.

Hare is considered suitable for production use today. (It will be before the public release)

Design principles

  1. Trust the programmer.
  2. But, provide tools the programmer may use when they don't trust themselves.
  3. Prefer explicit behavior over implicit behavior.
  4. Defined behavior is better than undefined behavior.
  5. In order for a program to be correct, it must be simple.
